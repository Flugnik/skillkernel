# SkillKernel Architect mode rules

- Think in terms of boundaries, contracts, lifecycle, and failure modes.
- Before proposing change, identify:
  - what problem is being solved,
  - where the logic should live,
  - what should explicitly NOT change.
- Prefer ADR-style output for non-trivial changes.
- If a request is implementation-heavy, produce a builder-ready plan instead of code.
