# new_skill_scaffold

## 1. Назначение
Этот skill задаёт порядок создания каркаса нового Kernel Skill внутри SkillKernel.

Его задача — чтобы новые skills создавались единообразно, с обязательной структурой, понятными файлами и минимально достаточным стартовым набором.

---

## 2. Когда применять
Применяй этот skill, когда нужно:
- создать новый Kernel Skill в проекте;
- подготовить базовую файловую структуру;
- завести обязательные файлы и стартовые заглушки;
- оформить новый модуль так, чтобы его можно было дальше развивать по стандарту проекта.

---

## 3. Когда не применять
Не применяй этот skill, если:
- нужно доработать уже существующий skill;
- задача касается Roo Skills для IDE, а не Kernel Skills runtime-системы;
- требуется не каркас, а полноценная реализация сложной логики;
- структура skill уже создана и нужна только локальная правка.

---

## 4. Входные условия
Перед началом должно быть понятно:
- имя нового Kernel Skill;
- его назначение;
- где он будет лежать в структуре проекта;
- какой минимальный набор файлов считается обязательным;
- нужен только scaffold или ещё и первичная логика.

---

## 5. Порядок действий

### Шаг 1. Зафиксируй имя и назначение
Определи, как будет называться новый skill и какую задачу он решает.

### Шаг 2. Создай минимальную структуру
Подготовь обязательные каталоги и файлы по проектному стандарту.

### Шаг 3. Добавь стартовые заглушки
Создай минимально понятные заготовки для manifest, plugin, тестов и других обязательных частей, если они входят в стандарт проекта.

### Шаг 4. Не уходи в лишнюю реализацию
Создавай каркас, а не полноценный продукт, если этого отдельно не просили.

### Шаг 5. Проверь пригодность каркаса
Убедись, что новый skill можно сразу взять в дальнейшую работу без переделки структуры.

---

## 6. Ограничения
Этот skill не должен:
- реализовывать весь skill целиком без отдельного запроса;
- смешивать scaffold с архитектурным рефакторингом;
- придумывать лишние файлы;
- нарушать существующий стандарт структуры SkillKernel.

---

## 7. Формат результата
Результат должен включать:
- список созданных файлов и папок;
- краткое назначение нового skill;
- указание, что именно уже создано, а что остаётся на следующую задачу.

---

## 8. Проверка перед завершением
Перед завершением проверь:
- структура создана;
- обязательные файлы на месте;
- имена согласованы;
- каркас минимален, но пригоден для дальнейшей работы;
- лишних сущностей нет.

---

## 9. Пример входа
- "Создай scaffold для нового skill order_intake."
- "Подготовь каркас skill для входной точки telegram_adapter."

---

## 10. Пример ожидаемого результата
- создана правильная структура каталогов;
- есть обязательные стартовые файлы;
- каркас готов для следующего шага реализации.
